Building a recommendation system¶

Possible approaches are the following:

Memory-based Collaborative Filtering -
This method calculates the similarity between users or items using the user’s previous data based on ranking. Predictions are calculated by applying statistical techniques to the entire dataset. To forecast what rating would a chosen user give to an item, we would find users similliar to them who have rated this item.

User-based filtering (alternative to item-based) goes like this:

  1. choosing other users that are similar to our user U (using cosine similiarity or other)
  2. predicting rating of the item I to a user U by calculating the results of aggregating similar users
  3. providing recommendations based on the results predicted in stage 2

Advantages: it is easy to implement and new data is easily accommodated. However, memory-based collaborative filtering has decreasing performance in data with high sparsity (this is the case for the Book-Crossing dataset) and have limited scalability for large datasets.

Model-based collaborative filtering -
This method uses machine learning algorithms to predict ratings of yet unrated products by customer. These algorithms are further divided into different subsets, i.e., Matrix factorization-based algorithms , deep learning methods, and clustering algorithms.

Advantages: we extract some information from the dataset, and use that as a "model" to make recommendations without having to use the complete dataset every time, therefore not as much memory is needed. Also, models tend to give more accurate predictions.

def similiarity(user1,user2):
    """ Calculates similarity score of two user. Chosen similarity metrics is cosine metrics.
        Args :  
            user1, user2 (float): two chosen user IDs
        Returns: 
            score(float): calculated similarity, in the range (0,1), where 1 is the most similar.            
    """
    df = users_books_ratings_me_added[['user_id', 'isbn', 'book_rating']] 
    df = df[(df['user_id']==user1) | (df['user_id']==user2)].fillna(0)
    
    if not user1==user2:
        df_user1 = df.copy()
        df_user1['book_rating'].loc[df_user1['user_id']==user2] = 0

        df_user2 = df.copy()
        df_user2['book_rating'].loc[df_user1['user_id']==user1] = 0
    else:
        df_user1 = df.copy()
        df_user2 = df.copy()
        
    a =  np.array(df_user1['book_rating'])
    b =  np.array(df_user2['book_rating'])
    
    score = dot(a, b)/(norm(a)*norm(b))
    
    return score

def get_top_n(predictions, n=10):
    """Return the top-N recommendation for each user from a set of predictions.

    Args:
        predictions(list of Prediction objects): The list of predictions, as
            returned by the test method of an algorithm.
        n(int): The number of recommendation to output for each user. Default
            is 10.

    Returns:
    A dict where keys are user (raw) ids and values are lists of tuples:
        [(raw item id, rating estimation), ...] of size n.
    """

    # First map the predictions to each user.
    top_n = defaultdict(list)
    for uid, iid, true_r, est, _ in predictions:
        top_n[uid].append((iid, est))

    # Then sort the predictions for each user and retrieve the k highest ones.
    for uid, user_ratings in top_n.items():
        user_ratings.sort(key=lambda x: x[1], reverse=True)
        top_n[uid] = user_ratings[:n]

    return top_n

def recommended_list(user):
    """
    Args:
        user: The user ID from the dataset.
    Returns:
        List of book titles from the unsimplified users_books_ratings dataframe
    """
    recommended_list = defaultdict(list)
    
    top_n = get_top_n(predictions, n=10)
    
    for i in top_n[user]:
        book, rating = i
        title = users_books_ratings_me_added[users_books_ratings_me_added['isbn']==book]['book_title'].unique()[0]
        recommended_list[title].append(rating)
    return recommended_list
